# SNU AI Challenge - Project Setup

## 1. Project Overview & Task Definition
### Task Definition: 텍스트로 풀어보는 장면의 재구성
주어진 스토리라인(캡션)에 맞게 4개의 이미지 프레임을 올바른 순서로 재배열하는 문제를 해결해야 합니다.

### 문제 배경 및 중요성
이 과제는 이미지와 캡션을 개별적으로 인식하는 수준을 넘어, 여러 장면(frames)을 스토리 라인(캡션)의 맥락 속에서 재구성하여 올바른 시간적 전개 순서를 복원하는 멀티모달 이해 능력을 평가합니다.

### 입력(Input) 및 출력(Output) 형식
* **입력**: 자연어 문장과 여러 장의 프레임으로 구성된 데이터
  * 예: `{ "text": "자연어 문장", "frames": [image_3, image_1, image_4, image_2] }`
* **출력**: 정답 순서대로 다시 배열하였을 때 각 프레임의 위치
  * 예: `[3, 4, 1, 2]` (정답 순서대로 다시 배열하였을 때 첫 번째 프레임은 3번째에 위치, 두 번째 프레임은 4번째에 위치, ...)

## 2. Dataset

본 대회는 비디오 프레임의 시간적 순서를 예측하는 과제입니다. 참가자는 뒤섞인 4개의 비디오 프레임 이미지와 해당 비디오에 대한 텍스트 설명(Sentence)을 바탕으로, 원본 비디오의 올바른 시간 순서를 복원해야 합니다.

데이터셋은 학습용(train)과 평가용(test)으로 구분되어 제공됩니다. 각 샘플은 고유한 Id를 가지며, 이에 대응하는 이미지 폴더와 메타데이터가 CSV 파일로 제공됩니다.

### 2.1 파일
* `train.csv`: 모델 학습을 위한 데이터. 비디오 프레임의 파일명, 텍스트 설명, 정답(원래 순서) 포함
* `test.csv`: 순서 예측을 수행해야 하는 평가용 데이터. 정답은 제공되지 않음
* `sample_submission.csv`: 제출 파일의 예시 포맷
* `train/`: 학습용 이미지 폴더. 각 폴더명은 `train.csv`의 Id와 일치
* `test/`: 평가용 이미지 폴더. 각 폴더명은 `test.csv`의 Id와 일치

### 2.2 CSV 컬럼

**train.csv**
* **Id**: 각 비디오 샘플의 고유 식별자 (영문 대소문자 및 숫자 조합 6자리)
* **Sentence**: 해당 비디오 클립을 묘사하는 텍스트 캡션. 프레임의 순서를 추론하는 데 중요한 단서
* **Input_1 ~ Input_4**: 모델에 입력될 4개 프레임의 이미지 파일명
  * 주의: 각 파일명은 랜덤한 3자리 영문 코드로 암호화되어 있으며, 파일명 자체에는 순서 정보가 없음
  * 이미지는 알파벳 순서로 정렬되어 제공되지만, 이는 실제 시간 순서와 무관함
* **No_ordering**: 프레임 재배열이 불필요한 데이터인지 여부 (True일 경우 이미지는 셔플링되지 않았으며, 정답은 `[1, 2, 3, 4]`로 고정)
* **Answer**: 프레임의 실제 시간적 순서를 나타내는 정답 라벨
  * 형식: `[n, n, n, n]` 형태의 리스트 문자열, n은 1~4의 정수 값
  * 예시: `[2, 4, 3, 1]`인 경우, 제공된 이미지 중 첫 번째(Input_1)가 실제로는 2번째 프레임, 두 번째(Input_2)가 4번째 프레임임을 의미

**test.csv**
학습 데이터와 유사하지만, 정답과 관련된 정보(`No_ordering`, `Answer`)가 제외됨
* **Id**: 비디오 샘플 고유 식별자
* **Sentence**: 비디오 텍스트 캡션
* **Input_1 ~ Input_4**: 암호화된 이미지 파일명

* 상금은 대표자 명의로 된 국내 은행 계좌로만 지급 가능

* 신청 후 팀 결성/분리/합병 불가능 (적발 시 실격)
* 동일인의 다계정 참가 금지 (적발 시 팀 전체 실격)

### 4.3 모델 학습·추론 및 외부 자원 사용 규칙
* **사용 환경**: Python, 모든 모델은 인터넷이 차단된 로컬 환경(CPU 또는 GPU)에서 실행 가능해야 함.
* **외부 API**: 
  * 학습 및 추론 과정에서 외부 상용 API(예: ChatGPT, Gemini, Grok 등) 사용 금지.
  * 데이터 전처리 목적에 한해 외부 상용 API 사용을 허용하되, **총 사용 비용은 3만 원을 초과할 수 없음**.
  * 외부 API 활용 시 발표 자료에 사용 방식, 실험 결과 및 발생 비용을 명시해야 함.
* **데이터 및 모델**:
  * 모델 학습 시 외부 데이터 사용 금지 (제공된 학습 데이터만 사용 가능).
  * 오픈소스 모델(예: LLaMA 등)은 **2026년 5월 31일 이전에 모델 가중치가 공개된 경우**에 한해 허용.
  * 모델 앙상블(여러 모델의 추론 결과를 조합하는 행위)은 허용하지 않음.
  * 모델 경량화 기법(Quantization, LoRA 등) 및 추론 전략(Chain-of-Thought, Multi-turn Chat, TTA 등)은 추론 시간 제약 내에서 허용.

## 5. Evaluation (평가 방법)
* **평가 지표**: **Exact Match Accuracy** (부분 점수 없음)
* 제출한 이미지 순서가 정답 순서와 완전히 동일한 경우에만 정답으로 인정되며, 순서가 하나라도 다를 경우 오답으로 처리됩니다.
  * 예: 정답이 `[1, 4, 2, 3]`일 때, `[1, 4, 2, 3]`만 정답으로 처리되며 그 외의 모든 순서는 오답으로 간주됩니다.
* **예선 리더보드**: 예선기간 중 리더보드는 전체 테스트 데이터의 70% (Public data)만을 이용하여 업데이트되며, 순위는 Exact Match Accuracy를 기준으로 결정됩니다.
* **본선 진출 및 검증**: 예선 종료 후 테스트 데이터 전체 (Public + Private data)에 대한 모델 성능 기준 상위 순위팀을 대상으로 코드 검증 및 보고서 검토를 수행합니다.
* **추가 평가**: 참가팀들의 성능 달성 수준에 따라 예선 종료 후 외부 데이터셋을 기반으로 한 성능 평가가 별도로 진행될 수 있습니다.

## 9. 부록: 개발 시 주의사항 (세팅·모델링 담당 추가)

* **출력 역변환 주의**: 모델이 "시간순으로 나열한 이미지 번호"(예: `[4, 2, 1, 3]`)를 출력하면, 제출 형식인 "각 이미지의 원래 위치"(예: `[3, 2, 4, 1]`)로 **역변환**해야 함 (베이스라인 `parse_model_output()` 참고)
* **로컬 검증 우선**: 제출은 팀 전체 1일 2회 → 실험 검증은 고정 평가셋 `splits/holdout_300.csv`(Exact Match 채점)으로, 제출은 확인용으로만. ⚠️ 이 평가셋의 Id는 파인튜닝 학습 데이터에서 반드시 제외
* **추론 시간 예산**: 24시간 / 819샘플 ≈ 샘플당 105초 이내 (현재 zero-shot 기준 0.7~1.7초로 여유)
* **오프라인 리허설**: 제출 전 `HF_HUB_OFFLINE=1` 환경에서 정상 실행 확인 (인터넷 차단 검증 환경 대응)
* **재현성 요건은 처음부터 준수**: `.py` 스크립트 + 상대 경로 + UTF-8 인코딩 + 전체 80GB 이하 + README (상위 16팀 코드 검증 대비)
* **현재 진행 상태·실행 방법·함정 목록**: `HANDOVER.md` 참조, zero-shot 비교 결과: `outputs/zero_shot_report.md`
