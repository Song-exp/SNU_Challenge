# SNU AI Challenge - Project Setup

## 1. Project Overview & Task Definition
### Task Definition: 텍스트로 풀어보는 장면의 재구성
주어진 스토리라인(캡션)에 맞게 4개의 이미지 프레임을 올바른 순서로 재배열하는 문제를 해결해야 합니다.

### 문제 배경 및 중요성
이 과제는 이미지와 캡션을 개별적으로 인식하는 수준을 넘어, 여러 장면(frames)을 스토리 라인(캡션)의 맥락 속에서 재구성하여 올바른 시간적 전개 순서를 복원하는 멀티모달 이해 능력을 평가합니다.

### 입력(Input) 및 출력(Output) 형식
* **입력**: 자연어 문장과 여러 장의 프레임으로 구성된 데이터
  * 예: `{ "text": "자연어 문장", "frames": [image_3, image_1, image_4, image_2] }`
* **출력**: 정답 순서대로 다시 배열하였을 때 각 프레임의 위치
  * 예: `[3, 4, 1, 2]` (정답 순서대로 다시 배열하였을 때 첫 번째 프레임은 3번째에 위치, 두 번째 프레임은 4번째에 위치, ...)

## 7. 프로젝트 고도화 및 전략적 방향성 (Improvement Roadmap)

대회 규칙(모델 앙상블 금지, 단일 RTX 3090 GPU 사용, 외부 API 사용 3만원 한도 등)을 고려할 때, 성능 극대화를 위한 핵심 전략은 다음과 같습니다.

### 7.1 프롬프트 고도화 및 추론 전략 (Prompt Engineering & Test-Time Strategy)
* **Chain-of-Thought (CoT) 도입**:
  * 모델에게 단순히 정답 리스트만 바로 출력하게 하지 않고, **"각 이미지 프레임의 내용을 설명 -> 캡션 속의 시간적 전개 선후 관계 분석 -> 최종 순서 도출"** 단계를 거쳐 생각하도록 유도합니다.
  * Qwen2-VL과 같은 시각 언어 모델은 CoT를 적용할 때 정답률이 크게 상승합니다.
* **퓨샷(Few-shot) 예시 제공**:
  * 훈련 데이터 중 직관적이고 대표적인 샘플 1~2개를 프롬프트에 예시로 포함하여 학습시킵니다.
* **TTA (Test-Time Augmentation)**:
  * 단일 모델 앙상블은 불가능하지만, 하나의 모델 내에서 입력 이미지들의 순서를 무작위로 섞은 프롬프트를 여러 번 입력한 뒤 각 결과의 투표(Voting)나 확률값 합산을 통해 최종 결과를 예측하는 TTA 방식은 허용되므로 성능을 안정화할 수 있습니다.

### 7.2 모델 스케일업 (Model Scale-up)
* 베이스라인은 `Qwen2-VL-2B`를 사용하지만, 제출 제한 환경인 **RTX 3090 (24GB VRAM)**은 **`Qwen2-VL-7B`** 또는 **`Qwen2.5-VL-7B`**(2026년 5월 31일 이전 공개 모델)를 충분히 감당할 수 있습니다.
* 4-bit 또는 8-bit 양자화(Quantization) 기법을 이용하면 7B 크기의 멀티모달 모델을 로드하여도 VRAM이 충분히 확보되며, 2B 모델 대비 시각 인지 및 언어 추론 능력이 비약적으로 높습니다.

### 7.3 파인튜닝 (Fine-tuning via LoRA)
* 훈련 데이터셋이 **9,535개**로 상당히 크기 때문에, Zero-shot 추론에 머무르기보다는 **PEFT(LoRA) 기법**을 활용하여 모델을 이 데이터셋에 직접 파인튜닝하는 것이 가장 강력합니다.
* 이미지와 텍스트 간 순서 매핑 방식을 집중 학습시키고, 정답 출력 포맷(`[a, b, c, d]`)을 완벽히 강제할 수 있습니다.
* PEFT 라이브러리(예: LLaMA-Factory, PEFT)를 사용해 RTX 3090 단일 GPU에서 QLoRA를 통한 7B 모델 학습이 가능합니다.
