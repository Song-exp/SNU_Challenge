# SNU AI Challenge — 비디오 프레임 순서 예측 경진대회

> 뒤섞인 4개의 비디오 프레임 이미지와 텍스트 설명을 바탕으로 원본 비디오의 시간 순서를 복원하는 대회.
> 본 문서는 프로젝트 세팅, 대회 규정, 개발 시 준수 사항을 정리한 가이드입니다.

## 1. Task 정의

### 입출력 형식

| 구분 | 내용 |
|------|------|
| **입력** | 자연어 문장(Sentence) + 4장의 프레임 이미지 |
| **출력** | 정답 순서대로 재배열했을 때 각 프레임의 원래 위치 |

```
입력 예: { "text": "자연어 문장", "frames": [image_3, image_1, image_4, image_2] }
출력 예: [3, 4, 1, 2]
  → 1번째 이미지는 원래 3번째, 2번째 이미지는 원래 4번째 프레임
```

⚠️ **출력 형식 주의**: 모델이 "시간순으로 나열한 이미지 번호"(예: `[4, 2, 1, 3]`)를 출력하면,
제출 형식인 "각 이미지의 원래 위치"(예: `[3, 2, 4, 1]`)로 **역변환**해야 함.
(베이스라인의 `parse_model_output()` 참고)

### 데이터셋

| 파일 | 내용 |
|------|------|
| `train.csv` | 학습용. `Id, Sentence, Input_1~Input_4, No_ordering, Answer` 포함 |
| `test.csv` | 평가용 (819 샘플). Answer 미포함 |

- 이미지 파일명은 랜덤 3자리 영문 코드로 암호화 → **파일명으로 순서 유추 불가**
- `No_ordering == True` → 셔플되지 않은 데이터, 정답은 `[1, 2, 3, 4]` 고정
- 이미지 경로: `{IMAGE_DIR}/{Id}/{Input_n 파일명}`

### 제출 파일 형식 (`submission.csv`)

```csv
Id,Answer
WI8o3F,"[3, 1, 4, 2]"
Ae7zLm,"[1, 2, 3, 4]"
```

## 2. 평가 기준

### 예선 (온라인 리더보드)

- **지표**: Exact Match Accuracy — 4개 위치가 **모두 일치해야만** 정답 (부분 점수 없음)
- 예선 기간 중 리더보드는 테스트 데이터의 **70% (Public)** 만으로 갱신
- 예선 종료 후 상위 팀 대상 전체(Public + Private) 코드 검증 및 보고서 검토

### 본선 (예선 점수 + 오프라인 발표)

| 평가 항목 | 점수 | 개발 시 고려사항 |
|---|---|---|
| 예선 점수 (추론 정확도) | 40 | 리더보드 성능 |
| 데이터 활용 | 15 | 전처리·증강 전략의 논리적 근거 문서화 |
| 모델 설계/학습 | 15 | 문제 특성(시간 순서)에 맞는 구조 선택 근거 |
| 최적화 방법론 | 10 | RTX 3090 24GB 제약에 맞춘 최적화 |
| 자원 효율성 | 10 | 메모리 사용량, 추론 지연 시간 측정·기록 |

| **총합** | **100** | |

> 💡 정확도(40점) 외에 **효율성/비용 관련 항목이 30점** → 실험 시 VRAM 사용량, 샘플당 추론 시간, 학습 GPU-hours를 항상 기록해 둘 것.

## 7. 베이스라인 현황 & 개선 방향 메모

- **베이스라인**: `Qwen/Qwen2-VL-2B-Instruct` zero-shot 프롬프팅 (학습 없음)
- 파싱 실패 시 `[1, 2, 3, 4]` 폴백 — `No_ordering` 데이터 덕에 기본 점수 확보 가능

- 검증 서버 기준 24시간 / 819샘플 ≈ **샘플당 105초 이내** 유지 필요 (여유 있게 잡을 것)
