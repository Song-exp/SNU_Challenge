# SNU AI Challenge - Modeling & Tuning Strategy Guide

본 가이드는 이미지 프레임 재배열 멀티모달 태스크를 성공적으로 해결하기 위한 탐색적 데이터 분석(EDA), 모델링, 하이퍼파라미터 및 프롬프트 튜닝 전략을 정리한 문서입니다.

---

## 1. EDA (탐색적 데이터 분석) 전략

기본 데이터 통계를 넘어, 모델 설계 및 전처리 규칙 제정에 직접적인 단서를 주는 실전형 분석을 지향합니다.

### 1.1 시간적 선후 관계 키워드(Temporal Keywords) 빈도 분석
* 캡션(Sentence)에 나타나는 대표적인 시간 관계 접속사/부사들을 추출하고 빈도를 측정합니다.
  * *대상 키워드*: `then`, `followed by`, `next`, `after`, `first`, `finally`, `second`, `lastly` 등
* **적용**: 학습 시 이들 단어에 중요도 가중치를 부여하거나 프롬프트 템플릿 설계 시 가이드라인으로 사용합니다.

### 1.2 프레임 간 유사도 및 연속성 분석
* 4개의 이미지 프레임이 동일 비디오에서 추출된 밀접한 시퀀스인지(배경 동일, 행동만 미세하게 변화), 아니면 다중 씬 전환(Scene Cut)이 일어나는 시퀀스인지 파악합니다.
  * *측정 지표*: 프레임 간 SSIM(Structural Similarity Index) 또는 픽셀 차이값 평균값 계산
* **적용**: 유사도가 높은 데이터(미세 행동 감지 모델 강점 필요)와 유사도가 낮은 데이터(전체적 맥락 분류 강점 필요)를 클러스터링하여 세부 모델 행동을 분기 처리할 수 있습니다.

### 1.3 캡션의 의미 단위 분할(Splitting)
* 영어 캡션은 대개 긴 복합문 형태입니다. 콤마(`,`), 세미콜론(`;`), `then` 등의 구분자를 기준으로 문장을 분할한 후 각 하위 단락(Sub-captions)이 각 이미지 프레임과 매칭되는지 분석합니다.

## 2. Modeling (모델링) 전략

대회 규정상 **앙상블(Ensemble)이 전면 금지**되므로, 단일 모델로서 최대의 표현력과 강인성을 확보할 수 있는 구조를 구축해야 합니다.

### 2.1 Vision-Language Model (VLM) 파인튜닝 (주력 접근법)
* **추천 베이스라인**: `Qwen2-VL-7B-Instruct` 또는 `Llama-3.2-11B-Vision-Instruct` (2026년 5월 31일 이전 공개 가중치 사용 조건 만족)
* **학습 방식**: 4장의 프레임 이미지를 시각적 컨텍스트로 제공하고, 캡션을 지시문(Instruct Prompt)과 함께 주입하여 모델이 정답 순열 `[3, 2, 4, 1]`을 텍스트 토큰 형태로 직접 출력하도록 파인튜닝합니다.
* **리소스 최적화**: 단일 RTX 3090 (24GB VRAM) 환경이므로 **QLoRA (4-bit 양자화 + LoRA)** 방식을 사용하여 메모리 점유율을 12GB 이하로 억제하면서 강력한 전이 학습을 수행합니다.

### 2.2 Dual-Encoder (CLIP) 코사인 유사도 랭킹 모델 (서브/속도 우선 접근법)
* 4개 프레임의 가능한 모든 순열($4! = 24$가지)에 대해 비디오 프레임 시퀀스 특징 벡터를 생성하고, 캡션 텍스트 임베딩과의 코사인 유사도가 가장 높은 순열을 정답으로 도출합니다.
* VLM 대비 계산량이 적어 추론 속도가 매우 빠르지만, 세부 행동 변화나 미세한 선후 관계 인지 능력은 다소 제한적일 수 있어 초기 프로토타이핑용으로 적합합니다.

## 3. Tuning & Optimization (하이퍼파라미터 및 프롬프트 튜닝)

### 3.1 프롬프트 및 추론 전략 튜닝 (Inference Tuning)
* **Chain-of-Thought (CoT) 포맷 제약**:
  * 모델이 정답 리스트를 생성하기 전, 각 프레임에서 일어나는 상황을 설명하고 캡션과의 인과적 시간 순서를 유추하도록 출력 형식을 튜닝합니다.
* **TTA (Test-Time Augmentation)**:
  * 프레임 입력 순서를 셔플링(예: [Image 3, Image 1, Image 2, Image 4])하여 여러 프롬프트로 다중 추론을 유도한 뒤, 투표(Voting) 혹은 Logits 앙상블을 수행하여 단일 모델 내에서 안정성을 강화합니다.

### 3.2 LoRA 파인튜닝 파라미터 튜닝
* **LoRA Target Modules**: 언어 레이어의 Attention 모듈(`q_proj`, `v_proj`)뿐만 아니라 Vision Encoder 및 Multi-modal Projector 레이어의 가중치들도 LoRA target에 포함시켜 결합 학습률을 높입니다.
* **LoRA Rank ($r$) & Alpha ($\alpha$)**: 과적합 방지와 표현력의 균형을 위해 `r=16, alpha=32` 또는 `r=32, alpha=64` 값에서 실험을 개시합니다.
* **학습률 스케줄러**: Cosine Annealing 스케줄러를 적용하고, 초반 불안정성을 방지하기 위해 3% 내외의 Warmup 비율을 설정합니다. (Learning Rate: `2e-5` ~ `1e-4` 구간 설정)

### 3.3 인프라 최적화 (Inference & Resource)
* **FlashAttention-2**: 이미지 주입으로 인해 증가한 토큰 시퀀스의 어텐션 연산 속도를 가속하기 위해 적용합니다.
* **vLLM Engine**: 최종 제출 모델 검증 시 24시간 추론 시간 제약을 넉넉하게 돌파하기 위해 고속 추론 서빙 엔진을 도입합니다.
