# 🛠️ Claude 피드백 기반 기하학적 물리 분석 보완 대책

이 문서는 Claude의 예리한 피드백(데이터 증강 편향, 데이터 클리닝 위험성, Windows VRAM 단편화, 피사체 추적 한계)을 해결하기 위한 **구체적인 구현 및 보완 설계안**입니다. 이 내용을 팀원(정현이 형)과 공유하여 학습 파이프라인에 즉시 반영할 수 있습니다.

## 📌 1. 데이터 증강 (Augmentation) 보완책

### A. 캡션 방향성 자동 스왑 (Caption Direction Swapping)
* **문제**: 이미지를 좌우 반전(Horizontal Flip)할 때 캡션의 이동 묘사(`left` ↔ `right`)를 수정하지 않으면 모순 데이터가 발생합니다.
* **보완안**: 데이터 로더(Dataloader) 또는 증강 스크립트 단에서 정규식을 이용해 캡션을 매핑합니다.
  ```python
  def swap_caption_direction(sentence):
      # 영문 및 국문 키워드 방향 교차 매핑
      mapping = {
          r"\bleft\b": "right", r"\bright\b": "left",
          r"\bleftward\b": "rightward", r"\brightward\b": "leftward",
          r"\bleftwards\b": "rightwards", r"\brightwards\b": "leftwards",
          "왼쪽": "오른쪽", "오른쪽": "왼쪽",
          "좌측": "우측", "우측": "좌측"
      }
      # 임시 플레이스홀더를 사용하여 교차 스왑 시 덮어쓰기 방지
      s = sentence.lower()
      # 예: left -> temp_right, right -> temp_left
      for k, v in mapping.items():
          s = re.sub(k, f"temp_{v}", s)
      for k, v in mapping.items():
          s = s.replace(f"temp_{v}", v)
      return s
  ```

### B. 쉬운 패턴 과적합 방지: 커리큘럼 학습 (Curriculum Learning)
* **보완안**: 물리 법칙이 100% 명확히 들어맞는 정합 샘플만 편식 학습하면 실제 거친 데이터(노이즈)에 취약해집니다.
  * **1단계 (Epoch 1~2)**: 물리 법칙 정합률이 높은 샘플(Clean subset) 위주로 학습하여 비디오의 순차적 흐름과 시간 관계의 기초를 잡습니다.
  * **2단계 (Epoch 3~5)**: 전체 데이터셋(노이즈 및 복잡한 물리 유형 포함)을 투입하여 실전 비디오 도메인의 복잡성에 적응시킵니다.

## 📌 2. 데이터 클리닝 및 노이즈 필터링 보완책

### A. 단순 탐지 실패와 실제 물리 법칙 위반의 분리
* **보완안**: 40%의 불일치 데이터를 일괄 삭제하지 않고, **탐지 실패**와 **진짜 모순**을 엄격히 나눕니다.
  1. **판정 불가 (Keep)**: 4개 프레임 중 객체가 2개 미만으로 검출되어 트래킹이 안 된 경우 $\rightarrow$ 이 데이터는 정상 학습용으로 **그대로 유지**합니다.
  2. **진짜 모순 (Soft-Weighting)**: 4개 프레임 모두 객체가 안정적으로 탐지되었으나 X좌표나 면적 흐름이 캡션과 정반대로 나오는 경우 $\rightarrow$ 이 샘플들은 학습에서 완전히 제거하는 대신 **Loss 가중치를 0.3~0.5배로 낮추어(Soft-down)** 잘못된 레이블 노이즈가 모델에 미치는 악영향을 억제합니다.

## 📌 4. 피사체 추적 한계 보완책 (Identity Tracking)

### A. CLIP Crop 거리 필터링 및 대규모 실측 검증 (A)
* **대규모 실측 검증 결과 (`scratch/calculate_real_crop_statistics.py` 수행)**:
  * holdout 데이터셋 300개 전수 중 피사체가 포함된 298개 샘플을 대상으로 OWL-ViT 및 CLIP을 적용해 **실제 크롭 이미지 분석**을 완료했습니다.
  * **실측 동일 피사체 쌍 (Real Same Subject Pairs, $N = 220$)**:
    * 평균 거리: **`0.2284`** | 표준편차: **`0.0960`** | 관측 최댓값: **`0.4943`**
  * **실측 타 피사체 쌍 (Real Different Subject Pairs, $N = 9097$)**:
    * 평균 거리: **`0.3775`** | 표준편차: **`0.0943`** | 관측 최댓값: **`0.6725`**
  * **지적 사항 실측 입증**: 전체 장면 전환용 임계값인 **`0.20`을 그대로 사용하면, 동일 피사체 추적 데이터의 70.8%를 "유실"로 잘못 판단(False Alarm)**하는 치명적인 노이즈 필터링 오차가 발생함을 실제로 확인했습니다.

* **📊 임계값 최적화 및 비대칭 비용 분석**:
  * 수학적 균형 정확도(Balanced Accuracy)가 최대화되는 최적의 경계값은 **`0.28` (정확도: 78.23%)**로 산출되었습니다.
  * 다만, 우리의 도메인 제약(정상적인 위치 궤적 정보를 오판하여 차단하는 False Alarm 최소화)에 따라 보수적인 임계값을 다음과 같이 튜닝합니다:
    * **`0.28` (수학적 최적)**: 오판정률(False Alarm) ~29.6%, 미검출률(Miss Rate) 15.1%
    * **`0.30` (기존 임계값)**: 오판정률(False Alarm) ~22.8%, 미검출률(Miss Rate) 20.6%
    * **`0.35` (권장 밸런스)**: 오판정률(False Alarm) ~10.3%, 미검출률(Miss Rate) 38.5%
    * **`0.38` (극보수적 안전선)**: 오판정률(False Alarm) ~5.7%, 미검출률(Miss Rate) 51.1%
* **보완안**: 이에 따라 본 프로젝트의 피사체 추적 마스킹을 위한 CLIP 크롭 이미지 거리 임계값은 정상 단서 차단율을 10% 수준으로 통제하면서도 오탐지를 효과적으로 걸러내는 **`0.35` (또는 극보수적 안전을 원할 경우 `0.38`)**를 잠정 최적값으로 정의하여 적용합니다.
  1. OWL-ViT가 검출한 각 BBox 영역을 크롭하여 CLIP 임베딩을 추출합니다.
  2. 인접 프레임 간 크롭 이미지의 CLIP 거리를 계산합니다.
  3. 만약 인접 프레임 간 물체 거리가 임계값 **0.35를 초과**하는 구간이 있다면, "동일 장면 내부에서 Tracker가 다른 사물로 미끄러졌다(Identity Jump)"고 판단하여 해당 궤적 힌트를 마스킹 처리합니다.
