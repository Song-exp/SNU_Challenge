## 1. 대회 태스크 및 제약 조건 개요
* **태스크**: 비작위로 뒤섞인 4장의 비디오 프레임(Image 1 ~ Image 4)과 1개의 비디오 캡션 문장(Sentence)이 주어졌을 때, 캡션 흐름에 맞는 올바른 시간 순서(예: `[3, 1, 4, 2]`)를 추론하는 태스크.
* **추론 단계 제약 조건 (중요)**:
  * 추론(Inference) 시 입력은 오직 **"테스트 문장 + 이미지 4장"**만 허용됩니다.
  * 추론 단계 프롬프트에 Gemma 라벨링 데이터, CLIP 이미지 유사도, OWL-ViT 탐지 좌표 등의 외부 파생 피처를 직접 텍스트 힌트 형태로 주입할 수 없습니다. (입력 분포 불일치 및 API 원격 로드 불가 제약 때문)
  * 따라서, 우리가 추출한 기하학/시공간 정보는 **(1) 학습용 데이터 증강(Augmentation) 선별**, **(2) 학습 데이터 노이즈 정제(Data Cleaning)**, **(3) 오프라인 성능 검증 및 오답 분석** 용도로만 활용해야 합니다.

## 2. 모델러(정현이 형)의 학습 파이프라인
* **모델**: `Qwen2-VL-2B-Instruct`
* **학습 방식**: LoRA SFT (Supervised Fine-Tuning)
* **주요 전략 (타겟형 증강 - exp16)**:
  * 전체 9,535개 비디오 중 가장 성능이 안 나오는 취약 유형(`sparse_camX`: 문장 내 사건 묘사가 2개 이하이고 카메라 묘사가 없는 평이한 영상, holdout 정확도 21.6%)을 선별하여 이 세그먼트의 데이터를 **4배 증강**하고, 나머지 데이터를 2배 증강하여 편향을 완화하는 학습 진행 중.
* **메모리 최적화 (Windows CUDA OOM 극복)**:
  * 윈도우 환경에서 학습 단편화(Fragmentation)로 인해 특정 스텝(540스텝)에서 지속적인 OOM 발생.
  * 이를 해결하기 위해 `train.py`에 **5종 메모리 위생 대책**을 적용:
    1. 마이크로 스텝마다 텐서 참조 즉시 해제 (`del loss`, `del outputs`)
    2. 50스텝마다 `torch.cuda.empty_cache()` 호출
    3. 모델 체크포인트 저장 후 즉시 캐시 청소
    4. VRAM 스파이크 발생 샘플 스킵 처리
    5. 학습 로그에 `reserved_vram_gb` 모니터링 출력

### D. 물리 인과성 가이드라인 설계 (V2 수정본)
* **수평 이동 물리 법칙의 방향성 교정**:
  * **카메라 패닝(Pan Left/Right)**과 **피사체 직접 이동(Move Left/Right)**은 X좌표 트렌드에 미치는 물리적 영향이 정반대이므로 분리 설계했습니다.
    * **Camera Pan Left** 또는 **Subject Move Right** $\implies$ 피사체는 우측으로 흐름 (**X 좌표 증가**)
    * **Camera Pan Right** 또는 **Subject Move Left** $\implies$ 피사체는 좌측으로 흐름 (**X 좌표 감소**)
  * **줌 인과성**: `zoom-in` $\implies$ 피사체 면적 증가, `zoom-out` $\implies$ 피사체 면적 감소.

## 4. holdout_300 데이터셋 전수 검증 결과 (V2)
이 수정된 물리 검증 스크립트를 300개의 고정 Holdout 데이터셋에 적용한 결과입니다.

* **평균 객체 검출 성공률**: 67.7% (4프레임 중 2.71프레임 성공)
* **평균 장면 그룹 수**: 2.97개 (CLIP 분할 결과)
* **물리 법칙 정합성 검증 결과**:
  * **줌(Zoom) 물리 일관성**: **60.3% (38/63)**
  * **수평 이동(패닝/직접이동) 물리 일관성**: **58.3% (14/24)** (기존 V1 버전의 부호 버그 수정 후 0.0%에서 비약적으로 상승)
  * **전체 물리 법칙 정합률**: **59.8% (52/87)**
  * 무작위 4프레임 정렬 성공 확률이 **4.1% (1/24)**인 점을 감안하면, **약 60% 수준의 물리 정합성 신호는 노이즈가 심한 비디오 데이터에서 강력한 Spatial-Temporal Prior로 기능**함을 입증합니다.
