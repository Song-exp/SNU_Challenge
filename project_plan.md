# 프로젝트 추진 계획: SNU AI Challenge 비디오 프레임 순서 예측 경진대회

본 문서는 PROJECT_SETUP.md의 대회 규정, 제약 조건, 평가 기준을 바탕으로, 기존의 `Qwen2-VL-2B-Instruct` 베이스라인(Zero-shot) 코드를 고도화하여 최종 제출 및 본선 발표까지 성공적으로 이끌기 위한 단계별 프로젝트 실행 계획입니다.

---

## 1. 현재 상황 분석 및 핵심 도전 과제

### 1.1 현재 상태 (As-Is)
- **모델**: `Qwen2-VL-2B-Instruct`를 활용한 Zero-shot 프롬프팅.
- **데이터**: `train.csv` (9,539개 샘플), `test.csv` (819개 샘플) 및 이미지 데이터가 로트 폴더에 분산 존재.
- **한계점**:
  1. **성능 제한**: 학습 데이터 9,539개를 전혀 활용하지 않고 Zero-shot으로만 추론하고 있어 Exact Match 정확도가 낮을 수 있음.
  2. **추론 속도**: CPU/오프로딩 환경에서 샘플당 약 150초 소요. 검증 서버의 제한 시간(819샘플 / 24시간 ≈ 샘플당 105초 이내)을 만족하기 위해 GPU 기반 배치 추론 전환 필수.

### 1.2 핵심 제약 사항 (Constraints)
- **하드웨어**: 단일 **GeForce RTX 3090 (VRAM 24GB)**. 학습 및 추론 시 VRAM 24GB를 초과하지 않아야 함.
- **인터넷 차단**: 최종 검증 서버는 폐쇄망 환경이므로 모든 가중치와 토크나이저가 사전에 다운로드되어 로컬에서 로드되어야 함 (`HF_HUB_OFFLINE=1`).
- **앙상블 금지**: 여러 모델의 예측값을 병합하거나 동일 모델의 다른 체크포인트를 조합하는 앙상블이 불가능하므로 **우수한 단일 모델**을 정교하게 학습시키는 것이 중요.
- **본선 평가 항목 (정확도 외 30점)**: 자원 효율성(VRAM 사용량, 추론 속도), 구축 비용(학습 연산량, API 비용)이 평가 대상이므로 **모든 실험에서 리소스 지표를 기록**해야 함.

## 2. 단계별 프로젝트 로드맵

### Phase 1: 로컬 개발 환경 구축 & 검증 전략 수립
> **목표**: 로컬에서 성능 향상을 정량적으로 측정할 수 있는 신뢰도 높은 실험 프레임워크 구축

1. **프로젝트 구조 표준화**
   - 권장 폴더 구조에 맞춰 `src/` 폴더를 생성하고 소스코드를 모듈화합니다.
   - `src/utils.py`, `src/train.py`, `src/inference.py`로 나누어 관리합니다.
2. **로컬 검증 데이터셋 (Validation Split) 구축**
   - 9,539개의 학습 데이터 중 일부(예: 10%, 약 950개)를 검증용으로 분리합니다.
   - `No_ordering` 변수(True/False)의 분포 비율을 유지하며 분리(Stratified Split)하여 실제 평가 환경과 유사한 검증셋을 구성합니다.
3. **오프라인 동작 환경 사전 테스트**
   - 환경 변수 `HF_HUB_OFFLINE=1` 및 `TRANSFORMERS_OFFLINE=1`을 설정하고 모델을 로드하여 완전히 독립된 환경에서도 실행되는지 코드를 미리 검증합니다.

### Phase 2: 데이터 분석 (EDA) 및 전처리
> **목표**: 비디오 프레임과 텍스트의 특성을 파악하고, 모델의 토큰 및 이미지 입력 효율을 최적화

1. **텍스트 및 이미지 EDA**
   - 이미지의 가로/세로 해상도 분포와 종횡비(Aspect Ratio)를 조사합니다.
   - 텍스트 설명(Sentence)의 길이 분포(단어 수, 문자 수)를 분석하여 모델의 맥락 길이(Context Length) 설정을 최적화합니다.
2. **이미지 리사이징 및 정규화**
   - Qwen2-VL 모델의 해상도 조절 가이드라인(`min_pixels`, `max_pixels`)에 맞춰 VRAM 사용량을 최소화하면서도 시각 정보 손실을 막을 수 있는 적절한 이미지 해상도 임계값을 설정합니다.
3. **상용 API 활용 데이터 정제 (선택 사항)**
   - 3만 원 이하의 비용 한도 내에서 텍스트의 맞춤법 교정, 특수문자 제거, 또는 비디오 순서 예측에 방해되는 노이즈 텍스트 필터링 등에 상용 LLM API를 부분 적용해 봅니다. (사용 내역 및 비용 기록 필수).

### Phase 3: 모델 선정 및 파인튜닝 (Fine-tuning)
> **목표**: LoRA/QLoRA 기법을 활용하여 단일 모델의 비디오 프레임 논리 복원 성능 극대화

1. **대상 모델 후보군**
   - `Qwen/Qwen2-VL-2B-Instruct` (경량 모델, 빠른 학습 및 추론)
   - `Qwen/Qwen2-VL-7B-Instruct` (VRAM 24GB 내에서 QLoRA 학습이 가능한 고성능 모델)
2. **QLoRA 파인튜닝 파이프라인 개발**
   - 단일 RTX 3090(24GB VRAM) 환경에서 7B 모델을 효과적으로 훈련하기 위해 `bitsandbytes`를 이용한 4-bit 양자화 및 LoRA(PEFT) 학습 코드를 구현합니다.
   - **학습 포맷 예시**:
     * **입력**: `Sentence: {자연어 문장}\nImage 1: <image_1>\nImage 2: <image_2>\nImage 3: <image_3>\nImage 4: <image_4>\n이 4장의 이미지의 원래 시간적 순서는 어떻게 되나요?`
     * **타겟**: `[3, 1, 4, 2]` (혹은 모델이 가장 자연스럽게 학습할 수 있는 텍스트 포맷)
3. **로컬 Metric 평가 및 최적 체크포인트 선정**
   - 학습 과정에서 Epoch별로 Validation Set에 대한 Exact Match(EM) 점수를 계산하고 가장 우수한 단일 체크포인트를 저장합니다.

### Phase 4: 추론 최적화 & 오프라인 검증
> **목표**: 24시간이라는 시간 제약 및 24GB VRAM 용량 제한 내에서 가장 빠르고 효율적인 추론 수행

1. **배치 추론 (Batch Inference) 구현**
   - 베이스라인의 Loop 방식(1개씩 추론) 대신, PyTorch DataLoader 및 HuggingFace의 패딩 기능을 적용하여 배치 단위(Batch Size 4~16)로 동시 추론하도록 `src/inference.py`를 최적화합니다.
2. **FP16/BF16 정밀도 적용 및 경량화**
   - 모델 가중치를 `bfloat16` 혹은 `float16` 정밀도로 로드하여 메모리 대역폭을 최적화하고 추론 지연 시간(Latency)을 단축합니다.
3. **안정성 확보 및 폴백(Fallback) 예외 처리**
   - 모델이 예측한 결과가 파싱할 수 없는 형식이거나 리스트 크기가 4가 아닌 경우, `No_ordering`이 True일 때의 정답인 `[1, 2, 3, 4]`로 안전하게 폴백되도록 예외 처리를 강화합니다.

### Phase 5: 최종 패키징 및 제출 보고서 작성
> **목표**: 예선 리더보드 순위 확보 및 본선 정성 평가(자원 효율성, 재현성) 만점 대비

1. **완전 재현성 검증**
   - 가상환경(`venv`)과 모델 가중치, 코드를 포함하여 로컬 실행 경로가 상대 경로로 올바르게 작동하는지 확인합니다.
   - 가중치 및 코드를 포함한 총 용량이 **80GB 이하**인지 최종 체크합니다.
2. **자원 및 비용 상세 로깅**
   - `logs/` 디렉터리에 각 실험 단계별 VRAM 사용량 Peak, 추론 시간(ms/sample), 총 학습 시간(GPU-hours) 및 API 사용 비용을 상세하게 문서화합니다.
3. **제출용 README.md 및 방법론 보고서(A4 5쪽 이내) 작성**
   - 제안하는 방법론의 논리적 근거(왜 QLoRA를 선택했는지, 전처리 전략의 의미 등)와 자원 효율화 수치들을 체게적으로 작성합니다.

## 3. 리스크 요인 및 대응 방안 (Risk Management)

| 리스크 요인 | 예상 영향 | 대응 방안 |
|---|---|---|
| **VRAM OOM (Out of Memory)** | 7B 모델 학습 시 GPU 메모리 부족으로 프로세스 중단 | 1) `per_device_train_batch_size`를 1로 낮추고 `gradient_accumulation_steps`를 늘려 가상 배치 크기 유지<br>2) 4-bit QLoRA 및 Gradient Checkpointing 활성화<br>3) 해결 불가능할 경우 `2B` 모델로 전환 후 훈련 |
| **모델 출력 파싱 오류** | 정답 포맷 불일치로 인한 점수 손실 | 1) 파인튜닝 시 정답 포맷 자체를 `[1, 2, 3, 4]` 형태로 출력하도록 유도 학습<br>2) 정규표현식 기반의 엄격한 후처리 필터 및 최종 `[1, 2, 3, 4]` 폴백 적용 |
| **오프라인 실행 오류** | 검증 서버 폐쇄망 환경에서 라이브러리 및 모델 가중치 누락으로 실격 | 1) 학습 완료된 가중치(LoRA merge 완료본)를 로컬 디렉터리에 완전히 저장<br>2) `HF_HUB_OFFLINE=1` 설정 하에 가상환경 내부 패키지만으로 추론 코드를 3회 이상 단독 실행 및 재현성 검증 |
| **시간 초과** | 819 샘플 추론 시간 24시간 초과 | 1) FlashAttention-2 또는 PyTorch `sdpa` 활성화<br>2) 배치 추론 속도 튜닝 및 이미지 해상도 임계값 다운사이징을 통해 안전선(샘플당 10초 이내) 확보 |

- **Step 2**: `train.csv`로부터 로컬 검증 셋(Validation Set)을 분리하여 오프라인 Exact Match 검증 파이프라인 완성하기.
- **Step 3**: 로컬 GPU(또는 코랩 환경)에서 학습을 진행하기 위해 `src/train.py` QLoRA 학습 스크립트 설계하기.
