# 🎬 SNU AI Challenge - 통합 검수기 프로그램 사용 가이드 (GUI)

이 폴더는 팀원들이 각자 분담된 이미지 및 문장 데이터(주어/서술어 개수, 통사 구조 유형, 모호성 검사)를 쉽고 빠르게 검수하고 실시간으로 저장할 수 있도록 제작된 **GUI 검수 프로그램**이 모여 있는 공간입니다.

---

## 📌 1. 실행 방법 (How to Run)

아래 방법 중 편한 방식으로 실행해 주세요.

* **방법 A (권장 — 원클릭 실행)**:
  * 본 폴더(`검수기프로그램/`) 내부에 있는 **`run_gui_inspector.bat`** 파일을 마우스로 더블 클릭합니다.
* **방법 B (VS Code / 터미널 실행)**:
  * VS Code 터미널(또는 CMD)을 열고 프로젝트 루트 폴더에서 아래 명령어를 실행합니다:
    ```bash
    python 검수기프로그램/gui_syntax_inspector.py
    ```

> [!NOTE]
> * **선행 조건**: 파이썬 환경에 `pandas`와 `Pillow(PIL)` 패키지가 설치되어 있어야 합니다.
> * **데이터 폴더 위치**: 프로젝트 최상위 폴더에 대회 이미지 폴더인 **`snuaichallenge_data/`** 폴더가 동일한 이름으로 들어있어야 합니다. (용량이 커서 깃허브에는 제외되어 있으니, 각자 다운로드받은 원본 데이터를 이 위치에 수동으로 배치해야 정상적으로 사진이 보입니다.)
> * 실행하면 검수자 이름 선택 창이 뜹니다. 본인의 이름을 정확히 클릭해 주세요.

## 👥 2. 팀원별 담당 범위 및 데이터 작동 방식

### ① 이름 선택 시 자동 범위 지정
검수기를 실행하고 자신의 이름을 선택하면, 프로그램이 각자에게 할당된 범위의 데이터만 자동으로 로드합니다.
* **병철**: `0` ~ `2999`행
* **서현**: `3000` ~ `6002`행
* **정현**: `6003` ~ `9534`행
> 각자 다른 범위에서 작업하므로, 서로의 작업 데이터를 침범하거나 덮어쓸 염려가 전혀 없습니다.

### ② 실시간 로컬 자동 저장 (복사본 방식)
* 프로그램을 처음 실행하면 프로젝트 폴더 루트에 원본 `train_검토_최종_완료.csv` 파일의 복제본인 **`train_검토_최종_완료_수정본.csv`** 파일이 자동으로 생성됩니다.
* 검수기 내에서 수치를 고치거나 모호함 체크박스를 건드릴 때마다, 이 **`수정본.csv` 파일의 수정 칼럼들에 실시간으로 즉시 저장**됩니다. 
* 작업 도중 프로그램을 언제든지 끄거나 창을 닫아도 안전하게 데이터가 저장됩니다.

### ③ 이어하기(Resume) 기본 지원
* 프로그램 종료 후 다시 켤 때 본인 이름을 선택하면, **가장 최근에 작업하다가 멈춘 지점(미완료된 첫 번째 행)부터 자동으로 다이렉트 연결**되어 이어서 작업할 수 있습니다.

## 🚀 3. 검수 속도를 3배 높여주는 핵심 기능

### 💡 추출 단어 실시간 시각화 (뇌정지 방지)
* 문장 전체를 보며 머릿속으로 주어와 동사를 일일이 찾아 세어볼 필요가 없습니다!
* 수치 입력 칸 바로 밑에 컴퓨터가 문장에서 추출한 **`추출 주어:`** 및 **`추출 서술어:`** 실제 단어 리스트가 실시간으로 표시됩니다.
* 단어 목록을 눈으로 슥 대조하고 맞으면 바로 **엔터(Enter)** 키를 쳐서 1.5초 만에 패스할 수 있습니다.

### ⌨️ 단축키 및 키 바인딩
* **수치 수정**: 키보드로 주어/동사 개수나 유형(Type) 입력.
* **`Enter` (엔터)**: 현재 수치 저장 및 **다음 사진/데이터로 넘어가기**.
* **`Backspace` (일반 화면 상태)**: 오타 확인 및 수정을 위해 **이전 사진/데이터로 돌아가기**.
  * *주의: `모호한 이유 / 메모` 텍스트박스 내부를 클릭하고 글자를 타이핑할 때는 백스페이스가 글자를 지우는 원래 기능으로 작동하며, 이전 페이지로 넘어가지 않습니다.*

## ⚠️ 4. 모호함 체크박스(⚠️) 작성 기준

검수 중 아래의 **2가지 케이스**에 해당한다면 주저하지 말고 **`⚠️ 애매함 / 모호한 샘플 체크`**를 누르고 옆에 짧은 이유(메모)를 기재해 주세요.

1. **텍스트-이미지 심각한 불일치 & 라벨 노이즈**:
   * 프레임 4개가 전부 똑같은 사진이라 시간 정렬이 무의미한 경우.
   * 문장의 묘사(예: "남자가 뛴다")와 이미지 프레임 내용(예: "자동차 주행")이 완전히 매칭되지 않는 경우.
2. **사람이 판단하기에도 문법/통사 구조 분류가 모호한 경우**:
   * Type-1, 2, 3 분류 기준이 애매하게 겹쳐 모호한 경우.

> [!IMPORTANT]
> **단순 장면 전환 횟수 오류 판정**:
> 컴퓨터가 판정한 장면 전환 횟수가 눈으로 본 것과 단순히 다른 경우(예: 실제론 2회인데 0회로 표기됨 등)는 **`모호함 체크(⚠️)`를 할 필요가 없습니다.** 수치 입력 칸의 **`수정된 장면 전환 횟수` 숫자만 올바르게 수정**해 주시면 됩니다. (학습에서 드롭할 필요가 없는 정상 데이터이기 때문입니다.)

> [!TIP]
> 체크된 모호한 샘플들은 최종 병합 후 **모델 학습 데이터셋에서 안전하게 제외(Drop)**하여 모델이 오염된 데이터를 학습하는 것을 방지하는 핵심 필터가 됩니다.

## 🔗 5. 작업 완료 후 최종 병합

1. 3명 모두 분담 영역 검수를 완료하면, 각자의 컴퓨터에 저장된 `train_검토_최종_완료_수정본.csv` 파일을 한 곳에 모읍니다.
2. 각자 담당한 범위가 겹치지 않으므로, 세 파일을 병합 스크립트에 넣고 실행하면 **수정 사항이 하나로 합쳐진 최종 검토본 CSV가 10초 만에 완벽히 생성**됩니다. (병합 스크립트는 검수 완료 후 AI에게 요청해 주시면 즉각 생성해 드립니다.)
