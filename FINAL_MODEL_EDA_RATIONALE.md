# 최종 모델 세팅 ↔ EDA 인사이트 매핑

> 우리 최종 8B 모델의 **모든 설정이 어떤 EDA/실험 발견에서 나왔는지** 추적한 문서.
> "왜 이렇게 세팅했나"를 근거와 함께 정리 (본선 보고서·팀 공유용).

---

## 한눈에 — 최종 모델

```
Qwen3-VL-8B (4bit QLoRA)
  + v5_reorder 프롬프트        ← 프롬프트 구조 실험
  + 타깃 증강 (sparse_camX 강조) ← 문장 유형 EDA
  + 무작위 셔플                ← 셔플 실험 (어려운셔플 기각)
  → 우도 K=4 순열 TTA (추론)    ← 위치 편향 EDA + 추론 레버
```

각 요소를 아래에서 근거와 함께 설명.

---

## 1. 모델: Qwen3-VL-8B (4bit)

**EDA/실험 근거:**
- **미니 스크리닝**: 2B→4B에서 **+13.5%p** (사상 최대 미니 신호) → 체급이 곧 성능
- **오답 해부 발견**: 주 오답 모드 = "쌍교환"(비슷한 두 프레임의 선후 혼동). 이건 **시각적 미세 구분력** 문제 → 파라미터 큰 모델이 유리
- **아키텍처 적합성**: Qwen3-VL은 interleaved M-RoPE + textual timestamp로 **비디오 시간정렬 특화** 설계 (기술보고서). 우리 태스크와 정확히 일치

**제약 반영:**
- 로컬 8GB는 8B 학습 불가 → **Kaggle T4×2(16GB×2)**에서 학습
- 채점환경 RTX 3090(24GB)이라 8B 제출 가능 (공개일 2025-10, 규정 통과)

---

## 2. 프롬프트: v5_reorder (문장 후치)

**개념**: 문장을 지시문 **뒤**(답 생성 직전)에 배치.
```
Look at the 4 images... Determine the correct order to match the sentence below.
Sentence: "{문장}"          ← 문장이 맨 뒤 (답 직전)
Provide the answer ONLY as a Python list...
```

**EDA/실험 근거:**
- **프롬프트 구조 실험 15종**: 화려한 문구(역할부여·스토리텔링)는 전부 노이즈. **문장 위치 이동만 유일하게 양성**
- 추론 스크리닝 +6, 미니 학습 +6.4%p, **풀 Public +0.9%p** (exp14 0.775 > exp07 0.766) — 3단계 전부 양성
- **원리**: 자기회귀 생성에서 문장이 답 직전에 오면, 이미지를 본 뒤 문장을 "신선한 상태"로 참조 → 매핑 정확도↑

**기각된 대안**: v4_story(역할부여)=노이즈, CoT 계열=4연패

---

## 3. 타깃 증강: sparse_camX 강조 ⭐ (EDA 핵심)

**개념**: 문장 유형별로 학습 반복 횟수를 다르게. **약점 유형을 더 많이 학습**.

**EDA 발견 (gemma 라벨 → 유형 분석):**
- **문장 유형화**: 비카메라 사건수(≤2/≥3) × 카메라표현(O/X) = **4유형**
- holdout 유형별 정확도:
  | 유형 | 정확도 |
  |---|---|
  | **sparse_camX** (사건적음+카메라없음) | **21.6% ← 최약점** |
  | sparse_camO | 48.7% |
  | dense_camX | 52.7% |
  | dense_camO | 73.8% |
- **camera 축**: 카메라 표현 유무가 +27.3%p (최강 인자)
- **무표지 축**: 시간표지 없으면 -40.5%p (단일 최강)

**세팅 반영:**
- `aug_weights` CSV로 **sparse_camX를 2배 강조** (나머지 ×1)
- 즉 "가장 못 맞히는 유형을 집중 학습"
- ⚠️ 이 가중 CSV가 EDA 인사이트의 **핵심 구현체** — aug_mult(단순 반복)와 별개로 유형별 강조를 담음

**중요**: max_pixels·항목수를 줄여 시간을 벌어도, **이 타깃 증강 비율은 유지** (EDA 인사이트 보존)

---

## 4. 무작위 셔플 (어려운 셔플은 기각)

**개념**: 증강 시 4프레임을 어떤 순서로 재배치하나.

**실험 근거:**
- **어려운 셔플 시도** (CLIP 유사쌍 순서 뒤집기): 오답 해부에서 쌍교환이 주 오답이라, 유사 프레임을 어렵게 배치하면 판별력↑ 기대
- **실측 결과**: holdout +0.4%p였으나 **Public -1.1%p** (exp20 0.846 < exp17 0.857) → **역효과**
- → **어려운 셔플 기각, 무작위 셔플 채택**

**교훈**: holdout↔Public 불일치 사례. 최종은 검증된 무작위 셔플.

## 5. 우도 K=4 순열 TTA (추론) ⭐⭐ (최대 레버)

**개념**: 생성(greedy) 대신 **24개 순열 후보를 전부 채점**하고, 이미지를 4가지 배치로 바꿔가며 점수 합산.

**EDA/실험 근거:**
- **위치 편향 발견**: 멀티이미지 VLM은 "먼저 보여준 프레임 = 시간상 먼저"라는 편향이 있음 → identity 지름길의 원인
- **오답 해부**: identity 답변([1,2,3,4] 그대로)이 주요 오답 모드
- **추론 레버 미개척**: 그동안 학습(증강·프롬프트)만 건드렸고, 추론 전략은 비어있었음

**작동 원리:**
1. 답 공간이 순열 24개로 닫혀있음 → 주관식(생성)이 아니라 **24지선다(채점)**
2. **greedy 근시안 회수**: 첫 토큰 확률 높아도 전체 순열 확률은 다를 수 있음
3. **순열 TTA (K=4)**: 4가지 배치로 채점→합산 → 위치 편향 상쇄

**실측**: holdout **52.38% → 57.14% (+4.76%p)** — 로컬 4B에서 나온 **최대 개선**
**규정**: 학습 없음, 입력 그대로(문장+이미지), TTA는 §4.3 명시 허용 → 완전 안전

## 6. 기각된 것들 (EDA로 죽인 카드)

| 시도 | EDA/실측 결과 | 왜 기각 |
|---|---|---|
| **CoT** (분석 후 답) | 4연패 (-4.4~-9.1%p) | 이미지 근거 없이 "정답을 길게 말하는 법"만 학습, identity 붕괴 |
| **이미지 힌트 주입** (scene_cuts/OWL-ViT 좌표) | v10 무효(0.855<0.857) | 저해상도라 작은물체 검출 실패, "유사하다" 힌트가 identity 증폭 |
| **gemma 힌트 입력** | 미니 2연패 | 힌트를 모델에 먹이면 무익 (선별·분석용으로만 가치) |
| **어려운 셔플** | Public -1.1%p | holdout↔Public 역전 |

**핵심 교훈**: gemma/CLIP/OWL-ViT 자산은 **"심판이지 선수가 아니다"** — 어떤 데이터를 강조할지(타깃증강) 결정하는 데만 쓰고, 모델 입출력엔 안 넣음.

| EDA 발견 | 최종 세팅 반영 |

## 8. gemma/CLIP 자산의 최종 역할

EDA에서 만든 보조 자산이 최종 모델에 **간접적으로** 기여:
- **gemma 라벨** → 문장 4유형 분류 → sparse_camX 식별 → **타깃 증강 CSV** (aug_weights)
- **CLIP scene_cuts** → 오답 유형 분석 (모델엔 미주입)
- 이 자산들은 **추론 파이프라인엔 안 들어감** → 규정상 단일 모델(Qwen+LoRA), 앙상블 논란 없음
