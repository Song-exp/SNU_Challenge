# 📁 VLM 순서 정렬 데이터 정합성 검토 가이드

> 💡 **검토 시작 전 준비물**: 
> 1. `train_검토_완료.csv` (기본 검토용)
> 2. `snu_clip_features.csv` (장면 전환 불일치 분석용)
> 3. 이미지 프레임 파일 (`Input_1` ~ `Input_4`)

---

## 🔍 검토 체크리스트 (3단계)

### 1단계. 📝 문장 내 주어/서술어 검토
* [ ] CSV에 표기된 주어/서술어 **단어**와 **개수**가 문장 내용과 문법적으로 일치하는가?
* [ ] `[unspecified subject]`로 표기된 문장이 실제 주어가 생략된 구조가 맞는가?
* ⚠️ **오류 발생 시**: 검토 결과란에 **[구문 오류]** 표시 후, 올바른 주어/동사 기록.

### 2단계. 🖼️ 장면 전환 횟수 이미지 대조
* [ ] `Input_1` ~ `Input_4` 이미지 시퀀스를 시간 순서대로 보며 실제 장면 전환(Cut) 횟수를 직접 카운트한다.
* [ ] 직접 센 횟수가 CSV 파일의 `장면 전환 횟수`와 일치하는가?

### 3단계. 🚨 장면 전환 오류 시 CLIP 교차검토 예약
* [ ] **만약 전환 횟수가 실제 이미지와 일치하지 않는다면?**
  * 노션 테이블의 **`CLIP_Check`** 칸에 체크(`TRUE`) 표시.
  * 추후 `snu_clip_features.csv` 파일의 프레임별 유사도 점수와 대조하여 임계값 오차 원인을 분석하기 위함입니다.
  * 비고란에 `(예: 3-4번 프레임 전환 누락, CLIP 임계값 검토 필요)` 메모 기재.
