# 최종 모델링 큰 그림 — 구조 인지형 순서 예측 파이프라인

> 작성: 7/17 (gemma 라벨 train 전량 확장 결정 직후)
> 위치: 상위 비전 문서. 단기 실행 계획은 `PLAN_post_labeling.md`, 현황은 `HANDOVER.md` 참조.

## 한 문장 요약

**문장의 서사 구조(카메라·시점·주체·사건·표지)와 이미지의 시각 구조(장면 전환점)를 명시적인 신호로 만들어, ① 데이터를 약점에 맞춰 증강하고 ② 모델이 그 구조를 근거로 추론하게 만드는 파이프라인.** 지금까지의 "문장+이미지 → 직답" 블랙박스를 "구조 파악 → 구조를 근거로 정렬"의 2층 구조로 바꾸는 것.

## 왜 이 방향인가 (실측 근거)

지금까지의 실험이 수렴하는 결론: **모델의 성패는 문장이 구조 단서를 얼마나 주느냐에 달려 있다.**

- 카메라 표현 유무: 62.5% vs 35.6% (**+27%p**, 길이 통제 후에도 +16.6%p)
- 서술된 사건 수: 1개 16.7% → 5개 74.5% (**계단식**, camera보다 가파름)
- 시간 표지·문장 길이도 같은 방향

즉 모델은 "구조가 풍부한 문장"만 잘 풀고 있고, test는 그 약점 구간(카메라X 56%, 사건 1~2개 33%)이 과반. 큰 그림은 이 격차를 두 방향에서 동시에 공략한다: **데이터로 (1단계), 추론 방식으로 (2·3단계).**

## 전체 데이터 흐름

```
[준비 단계 - 오프라인]
train 9,535 문장 ──gemma4:12b──> 구조 라벨 (주체/사건/camera/viewer/표지)
4장 이미지      ──CLIP──────> 인접 프레임 유사도 → 장면 전환점 (몇 번째 사이가 큰 변화인지)
(test 819 라벨·피처는 추론 시 입력 전처리로만 생성·사용 — 학습 설계에 쓰면 규정 위반, 아래 참조)

[1단계 - 데이터 전략]
holdout(=train) 구조별 정확도 ──> "재학습 기준선": 정확도가 낮은 구조 조건 (= 모델의 약점)
                                   ↓
              해당 조건의 train 샘플에 증강 가중 (예: 카메라X aug4 / 카메라O aug2)

[2단계 - 학습]
학습 샘플 = 이미지 4장 + 문장 + [구조 정보] + 정답 순서
  · 구조 정보의 주입 방식 = A/B 두 변형 (아래 분기)
  · CLIP 전환점도 같은 방식으로 힌트에 합류

[3단계 - 추론]
문장·이미지 → 구조 파악 → "사건 k개가 전환점 위치와 어떻게 대응하는가"를
근거로 문장-이미지 정렬 → 최종 순서 출력
```

## 단계별 상세

### 1단계 — 구조 기준선 기반 타깃 증강

- **입력**: train 전량의 5축 구조 라벨 (gemma + 정규식 교차 검증, camera 축 일치율 92.1%)
- **로직**: **holdout에서 정확도가 낮은 구조 조건**(= 모델의 약점)을 찾아, 그 조건에 해당하는 train 샘플의 재셔플 증강 배수를 올림
- ⚠️ **규정 (7/17 확인)**: "평가 데이터 특성 분석 후 학습"은 명시적 실격 사유 (`PROJECT_SETUP.md` §4.3 데이터 누수). **증강 가중의 근거는 holdout 약점 분석만** — test 분포는 학습 설계의 어떤 결정에도 쓰지 않는다. camera·n_events 발견 자체가 holdout 실측이므로 전략 실손 없음
- **원칙**: 축은 검증된 것부터 하나씩 (camera → n_events → 이후). 교차 조건은 holdout 셀 크기가 유지될 때만 (5축 동시 교차는 셀당 표본이 부족해 기준선 자체가 노이즈가 됨). viewer 축은 역방향(-6.8%p, n=19) 잠정 발견이라 기준선에서 제외, train 라벨로 재확인 후 판단
- **1호 실험**: exp15 (카메라X aug4 / 카메라O aug2)

### 2단계 — 구조 정보를 아는 모델 만들기

핵심 아이디어: 모델이 답을 내기 전에 **문장의 구조 수치(주체·사건·표지 개수, 카메라·시점 여부)를 먼저 파악**하게 하고, 그 수치와 **CLIP이 찾은 이미지 전환점**을 정렬의 힌트로 쓰게 한다.

구조 정보를 모델에 넣는 방식은 두 변형이 있고, 미니 1000 실험으로 승자를 정한다:

| | A. 입력 힌트형 | B. CoT 생성형 |
|---|---|---|
| 방식 | 프롬프트에 구조 라벨을 명시적으로 제공 | 모델이 구조를 먼저 출력한 뒤 답 (gemma 라벨을 CoT 타깃으로 SFT) |
| 장점 | 정답 구조를 줌 (오류 전파 없음), 추론 빠름 | 추론 시 외부 모델 불필요 (규정 안전) |
| 단점 | test에도 라벨러 필요 → 규정 확인 필수 | 자가 생성 오류 전파, 추론 13배 비용 |
| 전례 | 무학습 주입은 무효 실측 → 학습 포함이 관건 | exp12 기각했지만 주범은 spacy 분해 품질(깨끗한 분해는 동률) → gemma 라벨로 재도전 |

CLIP 전환점 힌트는 승자 변형에 합류시킨다 — 문장 구조(사건 k개)와 이미지 구조(전환점 위치)를 잇는 **유일한 시각-텍스트 다리**로, exp12의 근본 결함([Visual Evidence] 부재)을 채우는 조각.

### 3단계 — 구조 근거 정렬 추론

최종 프롬프트의 목표: "사건이 3개고, 이미지 2-3 사이에 큰 전환이 있다 → 사건 1·2는 앞 이미지들, 사건 3은 뒤 이미지에 대응" 같은 **구조-구조 대응을 명시적 근거로 순서를 결정**하게 만드는 것. 직답("그냥 순서 맞혀")에서 근거 기반 정렬("구조가 이러니까 이 순서")로의 전환이 이 그림의 종착점.

## 최종 제출 형태 (규정 제약 반영)

- 규정상 단일 모델 → 최종 제출물 = **Qwen3-VL(2B 또는 4B) + 승자 레시피로 학습된 LoRA 어댑터 1개**
- gemma·CLIP은 **학습 데이터 제작(오프라인)에만** 쓰면 규정과 무관. **추론 시에도** 쓰는 A형·CLIP 힌트는 규정 확인 결과에 따라 포함/제외
- 각 단계는 독립적으로 기여 가능한 **폴백 계단**:
  1. 3단계가 다 무산돼도 1단계(타깃 증강)만으로 exp07(48.41%)을 넘는 게 1차 목표
  2. 2단계 승자가 나오면 그 위에 얹음
  3. CLIP 힌트는 마지막에 합류

## 규정 확인 결과 (7/17, `PROJECT_SETUP.md` §4.3 정독)

1. **test 분석 → 학습 설계 = 실격 사유** ("평가 데이터 특성 분석 후 학습" 명시 금지). 증강·실험 우선순위의 근거는 항상 holdout/train 실측으로 문서화할 것
2. **추론 시 전처리 모델(gemma 라벨러·CLIP)은 허용으로 판단** — 앙상블 금지의 정의가 "여러 모델의 *추론 결과* 조합"인데 피처·라벨은 태스크의 답이 아님. CoT·TTA 등 추론 전략 허용 조항과 정합. 24h/819샘플·80GB 제약도 여유. 단 어댑터 2개 라우팅은 추론 결과 조합이라 불가(기존 판단 유지)
3. **남은 확인 1건**: gemma4 가중치 공개일이 2026-05-31 이전인지 (A트랙을 추론 파이프라인에 넣을 때만 해당. 학습 데이터 제작 전용이면 무관하나 발표 자료에 사용 방식 명시 필요)

## 미검증 가정 (이 그림이 틀릴 수 있는 지점)

1. 약점 구간 증강이 실제로 그 구간 정확도를 올리는가 → **exp15가 판정**
2. 구조 정보가 "난이도 상관"을 넘어 "정렬에 쓸 수 있는 정보"인가 → **미니 A/B가 판정** (기준선 mini_v1_aug2 15.5%, +4%p↑만 확전)
3. CLIP 전환점이 학습에 포함되면 유효한가 (무학습은 무효 실측) → **CLIP 합류 미니가 판정**
